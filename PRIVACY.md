# Privacy Policy

## Data Handling

ad-health-checker does not collect, transmit, or store any personal data, user information, or telemetry.

## Local Operation

This tool runs entirely on the system where it is invoked. All checks are performed locally; no data is sent to external services or servers.

## Output Data

Output generated by ad-health-checker may contain:

- Domain and controller names you specify
- IP addresses of domain controllers
- Network connectivity status
- Service availability information

This information remains on your local system unless you explicitly save or transmit output files. When sharing diagnostic output:

- Use sample or example data in documentation and tests
- Do not commit real domain names or IP addresses to version control
- Sanitize output before sharing with third parties

## Sample and Fake Data

All examples in documentation and tests use fictional domain names and IP addresses:

- `corp.example.com` — example domain
- `corp.local` — example local domain
- `10.0.0.0/24` — example IP ranges (RFC 5737 documentation prefix)
- `dc1`, `dc2` — example controller hostnames

These are not real systems and exist only for illustrative purposes.

## No External Calls

This tool makes no external HTTP requests, DNS queries beyond your specified targets, or any communication with third-party services.
